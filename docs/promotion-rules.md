# Promotion Rules

The daily note is not enough by itself.

Use the daily note for active context and audit trace, then promote durable state when needed.

## Promote to `projects.md` when:
- a project starts, pauses, resumes, or completes
- the project direction changes
- the next step changes materially
- key files change in a way that affects future work
- the project state would matter across future sessions

## Promote to `decisions.md` when:
- a decision changes how future work should be done
- a durable interpretation becomes the governing rule
- a previously active decision is superseded, abandoned, or otherwise stops governing current behavior

## Keep in the daily note only when:
- the item is temporary
- the item matters mainly for the current day or active window
- the item does not change future cross-session behavior

## Audit rule
- When memory is promoted, add a short line to the current daily note saying what was saved, where it was saved, and why.
